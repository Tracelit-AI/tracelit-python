You are integrating the Tracelit Python SDK.

There is NO tracelit.init() API.

Choose the correct setup by app type:

DJANGO (do NOT call auto_start in settings.py):
  INSTALLED_APPS = ["tracelit.integrations.django.TracelitConfig", ...]
  Set TRACELIT_* environment variables (bridge decouple into os.environ if needed).
  pip install opentelemetry-instrumentation-django
  Celery workers: install_celery_hook() in celery.py

FLASK:
  In app.py, call tracelit.auto_start(...) BEFORE routes handle traffic.
  Expose module-level `app = Flask(__name__)`.
  app.wsgi_app = TracelitWSGIMiddleware(app.wsgi_app)
  pip install opentelemetry-instrumentation-flask opentelemetry-instrumentation-wsgi
  FLASK_APP=app.py flask run

FASTAPI:
  In main.py, call tracelit.auto_start(...) before serving.
  app = TracelitASGIMiddleware(FastAPI())
  pip install opentelemetry-instrumentation-fastapi opentelemetry-instrumentation-asgi
  uvicorn main:app

SCRIPTS / CLI / BATCH (no web framework):
  tracelit.auto_start(api_key=..., service_name=..., environment=...)
  or tracelit.auto_start() with TRACELIT_* env vars

Install opentelemetry-instrumentation-* packages for libraries in use.
Missing instrumentation packages are skipped silently.

Disable in tests: TRACELIT_ENABLED=false

Manual spans:
  with tracelit.tracer().start_as_current_span("op"):
      pass
